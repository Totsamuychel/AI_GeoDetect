# 🧠 План навчання моделі геолокації

Цей документ детально описує стратегію навчання нейромереж для визначення геолокації за вуличними фотографіями України та Східної Європи.

---

## 1. Аугментація даних (Data Augmentation)

Для того, щоб модель була стійкою до різних умов зйомки та не перенавчалася на конкретних кадрах, ми використовуємо багаторівневу систему трансформацій (файл `code/augmentations.py`):

### 🔄 Базові трансформації (Train)
*   **RandomResizedCrop**: випадкове вирізання ділянки зображення (70-100% площі) та зміна розміру до робочого (напр. 260x260). Це вчить модель не залежати від точного розташування об'єктів.
*   **RandomHorizontalFlip**: горизонтальне дзеркальне відображення. Подвоює об'єм даних. *Примітка: вертикальне перевертання не використовується, оскільки вуличні фото мають чітку орієнтацію «земля-небо».*
*   **ColorJitter**: випадкова зміна яскравості, контрасту, насиченості та відтінку. Допомагає моделі працювати в похмуру погоду, при яскравому сонці або через фільтри камер.
*   **RandomGrayscale**: конвертація в ч/б з імовірністю 10%. Вчить модель спиратися на архітектурні форми, а не лише на кольори знаків чи будинків.

### 💪 Посилена аугментація (Strong)
Для складних архітектур використовується **RandAugment** та **RandomErasing** (затирання частин фото), що змушує мережу шукати дрібні контекстні підказки (наприклад, форму стовпів або тип бордюрів).

### 🧐 Тестова аугментація (TTA)
Під час інференсу (передбачення) ми використовуємо **Test-Time Augmentation**: робимо 5 різних версій одного фото (дзеркалення, зсув), отримуємо 5 прогнозів і усереднюємо їх для максимальної точності.

## 2. Стратегія валідації (Validation)

Для об'єктивної оцінки якості використовується наступний підхід:

*   **Stratified Split (85/15)**: Датасет розділений так, щоб пропорція міст та країн у навчальній та валідаційній вибірках була ідентичною.
*   **Early Stopping**: Навчання автоматично зупиняється, якщо `val_loss` не покращується протягом 7 епох. Це запобігає "зазубрюванню" (Overfitting).
*   **Метрики**:
    *   **Top-1 Accuracy**: чи вгадала модель точне місто/комірку.
    *   **Top-5 Accuracy**: чи входить правильна відповідь у п'ятірку найбільш імовірних (критично для схожих міст).
    *   **Distance Error**: середня відстань у кілометрах між прогнозом та реальністю.

## 3. Три способи (архітектури) навчання

Ми використовуємо три різні техніки, від простої до найбільш сучасної:

### 🔹 Спосіб 1: Baseline (Класична CNN)
*   **Модель**: EfficientNet-B2.
*   **Підхід**: Класична задача класифікації. Ми розбиваємо територію на класи (міста або H3-комірки) і вчимо модель передбачати номер класу.
*   **Перевага**: Найвища швидкість роботи та легкість у розгортанні на мобільних пристроях.

### 🔹 Спосіб 2: StreetCLIP (Transfer Learning)
*   **Модель**: geolocal/StreetCLIP (на основі ViT-L/14).
*   **Підхід**: Використання "фундаментальної" моделі, яка вже бачила мільйони вуличних панорам світу. Ми заморожуємо основні шари (Backbone) і донавчаємо лише фінальний класифікатор (Linear Probe) на українських даних.
*   **Перевага**: Дуже висока точність навіть на невеликій кількості фото (наші 7700 фото для неї — ідеальний об'єм).

### 🔹 Спосіб 3: GeoCLIP (Контрастивне навчання)
*   **Модель**: CLIP Vision Encoder + GPS Location Encoder.
*   **Підхід**: Модель не просто вчить класи, вона вчить **координати**. Ми використовуємо Random Fourier Features для кодування (lat, lon) у вектори. Навчання йде через **InfoNCE Loss**: модель намагається "наблизити" вектор зображення до вектора його реальних координат у просторі та "відштовхнути" від усіх інших точок.
*   **Перевага**: Модель розуміє безперервність простору. Якщо вона помиляється, то зазвичай пророкує сусідню вулицю чи місто, а не інший кінець країни.
